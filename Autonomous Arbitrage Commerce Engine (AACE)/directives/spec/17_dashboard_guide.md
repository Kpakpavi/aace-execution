# AACE MVP Dashboard Guide

## 1. Purpose

This document is the operator and developer guide for the AACE MVP Dashboard.

It explains:

- how to use the dashboard as a user,
- how to understand each section and metric,
- how to navigate from the dashboard to deeper system views,
- how to interpret dashboard outputs in the context of the system.

This guide is written for users who need to operate the system, not just read data from it.

---

## 2. Who This Guide Is For

This guide is intended for:

- eCommerce operators using AACE to monitor price discrepancies,
- managers reviewing system activity,
- developers validating system behavior,
- admins diagnosing system state.

---

## 3. Accessing the Dashboard

The dashboard is a protected view.

To access it:

1. Log in with your AACE account credentials.
2. Navigate to the dashboard from the main navigation.
3. The dashboard will display summary data based on your role and permissions.

If you cannot access the dashboard, verify that your account is active and that you have the required role.

---

## 4. Dashboard Sections

### 4.1 Summary Metrics

This section provides a high-level view of system activity.

| Metric | What It Means |
|--------|--------------|
| Total Opportunities | Number of price discrepancy opportunities detected in the selected time window |
| Average Score | Mean opportunity score across all detected opportunities in the time window |
| Opportunities by Status | Breakdown of opportunities by their current review status |
| Total Alerts | Number of alerts generated in the selected time window |

Metrics reflect data that has been processed and scored. They do not include in-progress or failed pipeline runs.

### 4.2 Recent Opportunities

This section displays the most recent opportunities detected by the system.

Each row includes:

- **Opportunity ID**: the unique identifier for this opportunity
- **Score**: the opportunity's calculated score (higher = more significant discrepancy)
- **Product**: the product this opportunity relates to
- **Status**: the current review status of the opportunity
- **Detected**: when the opportunity was detected

To view full details of an opportunity, navigate to the opportunity review section.

### 4.3 Recent Alerts

This section displays the most recent alerts generated by the system.

Each row includes:

- **Alert ID**: the unique identifier for this alert
- **Opportunity**: the opportunity this alert relates to
- **Score**: the opportunity score that triggered the alert
- **Summary**: a brief description of the alert
- **Generated**: when the alert was created

Alerts are generated for opportunities that exceed a defined scoring threshold.

### 4.4 Navigation

The dashboard provides quick links to:

- **Opportunity Review**: view, filter, and review all detected opportunities in detail
- **Reports**: access detailed reporting views for opportunities, alerts, and price history

Links are only visible if your role permits access to those sections.

## 5. Understanding Metrics

### Time Window

All dashboard metrics are scoped to a time window.

The default time window is displayed above the metrics section.

Metrics reflect only the data within that window.

### Score Interpretation

Opportunity scores are calculated using deterministic rules defined in the scoring specification.

Higher scores indicate:

- larger price discrepancies,
- higher confidence in the opportunity signal,
- higher priority for review.

Score thresholds for alerts are defined in the alert notification rules.

### Status Values

Opportunities may have the following status values:

| Status | Meaning |
|--------|---------|
| New | Detected but not yet reviewed |
| Under Review | Being actively reviewed |
| Actioned | A decision was made |
| Dismissed | Reviewed and not pursued |

---

## 6. What the Dashboard Does Not Do

The dashboard is a read-only summary surface.

It does not:

- take automated actions on your behalf,
- reprice products,
- place marketplace orders,
- modify any system records.

To take action on an opportunity, navigate to the opportunity review section.

---

## 7. Troubleshooting

### Dashboard Shows No Data

Possible causes:

- no product data has been ingested yet,
- no price observations have been recorded,
- no discrepancies have been detected in the current time window,
- your role does not have access to the relevant data.

Check the system activity logs or contact an admin if data is expected but not appearing.

### Metrics Appear Incorrect

If dashboard metrics do not match your expectations:

- verify the time window being used,
- check the reporting section for more detailed breakdowns,
- confirm ingestion and scoring jobs completed successfully.

If incorrect data persists, log the issue and escalate following the incident response procedure in the runbook.

### Access Denied

If you cannot access the dashboard:

- confirm your account credentials are valid,
- confirm your account role permits dashboard access,
- contact an admin if access should be granted but is blocked.

---

## 8. Best Practices

- Review the dashboard at regular intervals to monitor system activity.
- Use the time window to focus on recent activity when investigating new signals.
- Navigate to the opportunity review section for detailed investigation of high-score opportunities.
- Use reporting views to analyze trends over longer periods.
- Do not rely solely on the dashboard for compliance or audit purposes. Use the audit log for authoritative event history.
